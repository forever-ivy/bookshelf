# 智慧借阅用户端 PRD

**版本：** v1.0  
**日期：** 2026-03-24  
**适用范围：** 用户端 App  
**对应定位：** 用户端是智慧借阅入口，管理员端是智能运营中枢

---

## 1. 产品概述

智慧借阅用户端是一款面向学生、教师和普通读者的校园借阅应用，核心目标是把“找书、借书、配送、归还、推荐”串成一条低门槛、高效率、可感知智能化的借阅链路。

相较于传统图书馆检索系统，用户端不只解决“有没有这本书”，更解决：

- 这本书适不适合我
- 这本书现在能不能借
- 我能不能不用跑去书柜，直接送到座位
- 借完以后怎么续借、归还、再推荐下一本

产品首版聚焦高校场景，依托智能书柜、库存系统、配送机器人和推荐引擎，为用户提供从检索到履约再到复借的完整闭环体验。

## 2. 产品定位

### 2.1 用户价值

为读者提供一个“像点外卖一样借书”的入口，降低图书发现成本、提升借阅便利性、提高图书流通效率。

### 2.2 平台价值

通过用户端沉淀用户画像、兴趣偏好、借阅行为和履约数据，反向驱动推荐策略、库存调度和书柜运营效率提升。

### 2.3 与管理员端分工

- 用户端：面向读者，负责需求发起、借阅决策、订单追踪、个性化推荐
- 管理员端：面向图书馆/书柜管理人员，负责库存、调度、履约、运营和数据分析

用户端只承接“发起与感知”，不承接“调度与配置”。

## 3. 目标用户

### 3.1 核心用户

- 学生用户：课程学习、考试复习、兴趣阅读是主要借阅动机
- 教师用户：教学参考、科研资料、课程推荐是主要借阅动机
- 普通用户：公共阅读、通识阅读和轻量借阅需求

### 3.2 用户分层

- 新用户：首次注册，需要快速完成身份绑定与兴趣初始化
- 高活跃借阅用户：重视找书效率、配送效率和续借提醒
- 目标明确型用户：带着书名、课程名或考试需求进入
- 兴趣探索型用户：依赖推荐、榜单和书单发现内容

## 4. 产品目标与核心指标

### 4.1 业务目标

- 提升图书被发现和被借阅的效率
- 提升配送/到柜取书的履约完成率
- 提升用户借阅频次与复借率
- 建立可持续迭代的个性化推荐基础

### 4.2 首版核心指标

- 注册到完成身份绑定转化率
- 搜索到详情页点击率
- 详情页到借阅/下单转化率
- 配送订单完成率
- 借阅完成率
- 7 日留存 / 30 日留存
- 推荐点击率 / 推荐借阅转化率
- 续借使用率
- 逾期率

## 5. 产品边界

### 5.1 本 PRD 覆盖内容

- 用户端账号与身份体系
- 图书搜索、发现、推荐、详情、借阅、配送、归还、个人中心
- 用户可见订单状态与借阅状态

### 5.2 本 PRD 不覆盖内容

- 管理员端库存管理、调度策略、机器人调度后台
- 图书编目、上架、运维配置
- 推荐模型训练与后台算法实现细节
- 机器人具体路径规划和硬件协议

### 5.3 前置依赖

- 后端具备图书、库存、用户、订单、推荐服务
- 管理员端维护库存与书柜状态
- 机器人配送能力和区域可用性已接入
- 校园身份体系或手机号登录体系可用

## 6. 核心使用场景

### 场景 1：我要快速找到一本书并借到

用户通过书名、作者、课程名或自然语言搜索图书，查看库存和可配送状态，并完成借阅。

### 场景 2：我不想去书柜，想送到座位

用户在图书详情页直接发起配送，选择座位或教室，查看预计送达时间和订单状态。

### 场景 3：我不知道读什么，希望系统推荐

用户基于兴趣标签、专业、课程、历史借阅记录获得推荐，并理解“为什么推荐给我”。

### 场景 4：我借了很多书，希望有人提醒我

用户在“我的借阅”里查看借期、逾期风险、续借入口和历史借阅记录。

### 场景 5：我看完书了，希望方便归还

用户发起还书或整理请求，选择归还点，查看机器人取回状态。

## 7. 信息架构

建议用户端 App 首版采用 5 个一级模块：

1. 首页
2. 找书
3. 借阅
4. 收藏
5. 我的

对应次级页面：

- 登录与身份绑定
- 首页
- 搜索结果页
- 图书详情页
- 配送/到柜下单页
- 我的借阅
- 智能推荐页
- 收藏与书单页
- 归还与整理请求页
- 个人中心
- 消息通知页
- 配送记录页

## 8. 功能需求

### 8.1 登录与身份绑定

#### 功能目标

完成用户登录、身份识别、基础画像建立，为推荐和借阅规则做准备。

#### 功能点

- 支持学号登录
- 支持工号登录
- 支持手机号登录
- 支持绑定学院、专业、年级
- 首次进入选择阅读兴趣标签
- 识别用户身份：学生、教师、普通用户

#### 业务规则

- 首次登录后必须完成身份绑定才进入首页
- 兴趣标签首次至少选择 3 个
- 身份识别结果用于推荐、借阅权限和活动内容配置
- 借阅额度、借阅时长、配送权益由后端规则控制，前端仅展示可见规则

#### 关键页面元素

- 登录方式切换
- 验证码/密码输入
- 身份信息表单
- 兴趣标签选择器
- 完成引导按钮

### 8.2 首页

#### 功能目标

让用户进入应用后能够快速完成“搜索、发现、借书、查看配送状态”。

#### 功能点

- 顶部搜索框
- 今日推荐
- 热门榜单
- 考试专区
- 一键借书入口
- 配送状态快捷入口

#### 内容模块建议

- 搜索框：支持书名、作者、关键词、自然语言输入
- 今日推荐：根据兴趣标签、专业、历史借阅生成
- 热门榜单：本周热门、本学院热门、考试专区
- 快捷操作：一键借书、查看订单、继续阅读、续借提醒

#### 业务规则

- 首页推荐内容支持个性化排序
- 榜单优先展示有库存或可调度库存的图书
- 有进行中订单时，配送状态入口前置

### 8.3 智能找书

#### 功能目标

帮助用户用最低认知成本找到“我想要的书”或“适合我的书”。

#### 功能点

- 按书名搜索
- 按作者搜索
- 按分类搜索
- 按课程名称搜索
- 按自然语言检索
- 支持模糊搜索和联想词

#### 搜索结果展示字段

- 封面
- 书名
- 作者
- 简介
- 馆藏状态
- 所在书柜
- 是否可立即配送
- 预计送达时间

#### 业务规则

- 搜索支持拼音、简称、错别字容错
- 自然语言搜索示例：
  - “机器学习入门”
  - “适合新手的心理学书”
  - “考研英语写作推荐”
- 默认优先展示可借且履约路径清晰的图书
- 当无现货可借时，展示相似推荐或最近可获得版本

### 8.4 图书详情页

#### 功能目标

让用户在一个页面内完成决策：值不值得借、怎么借、现在能不能拿到。

#### 功能点

- 图书基础信息
- 内容简介
- 目录
- 标签
- 推荐理由
- “借过这本书的人还借了什么”

#### 操作按钮

- 立即借阅
- 加入收藏
- 加入稍后阅读
- 请求配送
- 到柜自取

#### 业务规则

- 若当前无库存但可调度，优先展示“请求配送”
- 若图书仅支持书柜领取，则隐藏或置灰配送入口
- 推荐理由需可解释，例如：
  - 与你的专业课程相关
  - 与你最近借阅图书相似
  - 当前处于考试复习阶段

### 8.5 到座借阅 / 配送下单

#### 功能目标

让用户明确知道“怎么拿到书”“多久能拿到书”“现在订单进行到哪一步”。

#### 功能点

- 选择取书方式：到柜自取 / 机器人配送
- 选择配送地点：阅览室座位、教室、指定区域
- 展示预计等待时间
- 展示订单状态

#### 订单状态

- 待取书
- 书柜出书中
- 机器人配送中
- 已送达
- 已完成

#### 业务规则

- 仅在配送区域开放时可选择机器人配送
- 若配送不可用，自动回退到到柜自取
- 下单前必须校验：
  - 是否有库存
  - 用户是否具备借阅权限
  - 当前地点是否支持配送
- 状态变化需给用户明确通知

### 8.6 我的借阅

#### 功能目标

让用户持续管理借阅周期，降低遗忘、逾期和流失。

#### 功能点

- 当前借阅列表
- 借阅期限提醒
- 续借功能
- 逾期提醒
- 历史借阅记录
- 借阅偏好总结

#### 业务规则

- 当前借阅按到期时间排序
- 临近到期的图书高亮提醒
- 续借是否可用由后端规则决定
- 历史记录支持按时间、分类、课程标签查看

### 8.7 智能推荐

#### 功能目标

将“找书”从主动检索扩展为主动发现，提升复借率和阅读深度。

#### 功能点

- 个性化推荐
- 相似图书推荐
- 专业课程推荐
- 阶段性推荐
  - 期末复习
  - 论文写作
  - 兴趣阅读
- 推荐解释

#### 推荐维度

- 用户兴趣标签
- 专业/年级/课程信息
- 历史借阅与浏览行为
- 校园热点与阶段性场景

#### 业务规则

- 每一组推荐至少给出一句“为什么推荐”
- 推荐内容优先考虑当前可借可履约资源
- 推荐结果应允许刷新与反馈，例如“不感兴趣”

### 8.8 收藏与书单

#### 功能目标

为用户提供长期阅读管理能力，提升沉淀与回访。

#### 功能点

- 收藏图书
- 自建书单
- 系统生成书单
  - AI 入门书单
  - 考研专区
  - 新书推荐
  - 教师推荐

#### 业务规则

- 收藏支持从列表页和详情页直接添加
- 系统书单可被收藏或一键加入稍后阅读
- 自建书单支持命名、排序、移除

### 8.9 归还与整理请求

#### 功能目标

降低归还门槛，并为后续机器人整理能力预留入口。

#### 功能点

- 发起还书请求
- 选择归还点
- 查看机器人取回状态
- 家庭场景扩展：整理散落书籍任务

#### 业务规则

- 校园场景首版主打还书请求与归还点选择
- “整理散落书籍”作为扩展能力保留，不进入首版核心闭环
- 归还状态需与借阅状态联动更新

### 8.10 个人中心

#### 功能目标

让用户可以查看和维护自己的阅读画像、通知、偏好和履约记录。

#### 功能点

- 兴趣标签维护
- 阅读画像展示
- 借阅偏好分析
- 消息通知
- 配送记录
- 积分 / 阅读成就

#### 业务规则

- 阅读画像需可理解，不做纯算法术语展示
- 消息分为系统通知、借阅通知、配送通知、推荐通知
- 配送记录支持查看历史订单详情

## 9. 关键流程

### 9.1 首次使用流程

登录 → 身份绑定 → 兴趣标签选择 → 首页推荐初始化 → 开始搜索或浏览

### 9.2 借阅流程

搜索 / 推荐进入详情页 → 查看库存与可配送状态 → 选择借阅方式 → 下单成功 → 查看订单状态 → 完成借阅

### 9.3 配送流程

详情页点击请求配送 → 选择地点 → 系统校验 → 生成订单 → 书柜出书 → 机器人配送 → 用户确认送达

### 9.4 归还流程

发起还书请求 → 选择归还点 → 生成归还任务 → 机器人取回 / 用户放回 → 状态完成

### P0（首版必须上线）

- 登录与身份绑定
- 首页
- 智能找书
- 图书详情页
- 到柜自取 / 配送下单
- 订单状态跟踪

### P1（首版增强）

- 智能推荐完整体系
- 收藏与书单
- 推荐解释
- 借阅偏好总结
- 配送记录

### P2（后续扩展）

- 家庭场景整理请求

- 更复杂的社交化书单和共读机制

## 11. 非功能需求

### 11.1 性能

- 首页首屏应快速可用
- 搜索结果返回需控制等待时间
- 订单状态更新需接近实时

### 11.2 可用性

- 核心操作路径不超过 3 到 4 步
- 重要状态必须可理解、可追踪
- 异常时给出替代方案，例如“暂不支持配送，可到柜自取”

### 11.3 可扩展性

- 支持后续增加更多身份类型
- 支持新增更多配送地点类型
- 支持接入更多推荐策略和书单主题

## 12. 数据埋点建议

建议埋点覆盖：

- 登录方式选择
- 身份绑定完成
- 兴趣标签选择
- 首页模块点击
- 搜索发起与搜索结果点击
- 图书详情页曝光与按钮点击
- 借阅下单成功/失败
- 配送状态查看
- 续借点击
- 收藏点击
- 推荐点击与推荐借阅转化

## 13. 风险与待确认问题

- 学号/工号登录接入哪套校园认证体系
- 借阅额度、借阅时长、续借规则是否按身份差异化
- 配送地点的可配送边界如何定义
- 配送 SLA 是否对用户可承诺
- 自然语言检索由规则检索还是大模型检索增强实现
- 推荐解释文案由规则生成还是模型生成
- “家庭场景整理请求”是否放入当前校园版信息架构

## 14. 成功标准

当用户端达到以下状态，可视为产品首版跑通：

- 用户能顺利完成登录、身份绑定和兴趣初始化
- 用户能通过搜索或推荐找到目标图书
- 用户能完成到柜自取或配送下单
- 用户能清晰查看订单与借阅状态
- 用户能收到续借、逾期、配送等关键提醒
- 平台能通过用户端稳定沉淀借阅、推荐和履约数据
