# Order Detail Modal Redesign Design

**Date:** 2026-03-23

## Goal

把订单详情页从“边看边在右侧长表单里改”的布局，改成“主页面负责查看，操作通过按钮打开 modal”的结构，让页面更清晰，也补上明确的返回路径。

## Context

当前订单详情页把订单快照、状态修改、人工处理、重试和归还处理都堆在同一个页面里。结果是：

- 页面像配置后台，不像详情页
- 信息展示和处理动作互相抢注意力
- 缺少明确的返回入口
- 高频动作虽然都在，但不够聚焦

## Decision

采用“读写分离”的详情结构：

- 详情页只展示订单状态、配送信息、处理摘要和归还信息
- 页面顶部提供 `返回订单列表`
- 页面内提供一组动作按钮：
  - `修改状态`
  - `调整优先级`
  - `人工处理`
  - `重试订单`
  - `处理归还`
- 每个动作各自打开一个 shadcn `Dialog`

## Layout

### Hero

- 保留现有 `PageShell`
- 左侧显示订单编号和说明
- 右侧放返回按钮和主要动作按钮

### Content

- `当前状态`：借阅、配送、任务、机器人四个状态块
- `处理摘要`：优先级、重试次数、人工处理状态、异常原因
- `归还信息`：归还申请状态、备注、入库位置；没有数据时显示空态

### Modal Rules

- 一个 modal 只负责一类操作
- modal 内只保留必要字段
- 提交成功后关闭 modal，并刷新详情数据
- 页面主体不再出现大段可编辑表单

## Testing

- 订单详情测试改成通过按钮打开 modal 后提交
- 保留原有接口调用断言
- 验证返回链接存在
