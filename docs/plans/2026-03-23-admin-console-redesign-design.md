# Admin Console Redesign Design

**Date:** 2026-03-23
**Status:** Approved for implementation
**Scope:** `/Users/Code/bookshelf/bookshelf/.worktrees/admin`

## Goal

在不改动现有后台功能边界的前提下，重新设计整套 `admin` 管理端视觉与信息结构，让 `dashboard / books / inventory / orders / robots / alerts / readers / recommendation / analytics / system` 形成一套统一、克制、有设计感的运营工作台。

本次重设计不追求“更花”，而是要把当前偏通用 SaaS 的卡片堆叠感，收敛成更像图书馆运营编辑台的页面秩序。

## Approved Direction

用户确认采用 `C` 方案：

- 保留浅色底色体系
- 弱化蓝白玻璃感
- 移除通用 SaaS 卡片墙观感
- 强化留白、排版、分栏、数据带与工作台结构

## Visual Thesis

纸感浅底、深色排版、细线分隔、少量冷蓝强调，让后台看起来像“高端图书馆运营台”，而不是“标准化业务系统模板”。

## Content Plan

每个页面遵循统一三层结构：

1. `页头层`
   页面标题、简短说明、页面级动作、当前状态或数据新鲜度。
2. `工作层`
   页面真正的操作区，不以卡片为中心，而以表格、分栏、数据带、筛选条、队列、编辑区为中心。
3. `重点层`
   当前页面最重要的聚焦区域，例如热力图、队列、趋势、详情检视器、配置编辑器。

## Interaction Thesis

1. 页面切换采用轻量淡入和平移，像翻页，不做弹跳式进入。
2. 导航、筛选、数据行和工作区分组采用极轻 hover/highlight，不依赖厚重阴影。
3. `dashboard` 与 `analytics` 增加轻微的节奏型 reveal，让数据层次更清晰，但不做营销化动效。

## Design Principles

- 默认去卡片化，只有当区域本身是“一个独立交互容器”时才保留容器感。
- 页面先做“版式与层次”，再做“组件与装饰”。
- 所有页面都优先让人一眼识别“这里看什么、这里改什么、这里决策什么”。
- 颜色只承担状态、导航焦点和关键操作，不承担装饰任务。
- 保留浅色背景，但减少发光、毛玻璃、大面积虚化。
- 标题、表格、说明文字之间拉开更明确的字重与字号差。

## Global Art Direction

### Color

- 基底：纸感白、暖灰白、冷灰白
- 文本：深墨黑、次级深灰
- 强调：单一冷蓝
- 状态色：成功绿、警示琥珀、错误红，但只用于状态与告警

### Typography

- 页面标题：更紧凑、更有编辑感
- 正文与表格：保持高可读性，避免展示型字体影响密度
- 标签与说明：减少大写营销语气，统一成产品 UI 文案

### Surfaces

- 大多数区域改成“平面分区 + 细边界 + 微弱底差”
- 仅保留少量关键容器，例如详情面板、编辑器、告警处理区
- 默认不再使用“每块都一个大圆角卡片”的模式

### Motion

- 导航切页：短时长 opacity + y transform
- 表格与筛选：shared layout / highlight
- 重点模块：section reveal

## Global Layout Anatomy

### Sidebar

- 由“玻璃侧栏”改为“安静的浅色书脊式导航”
- Logo、产品名、导航分组、权限入口更克制
- 激活态采用细长高亮条与浅底色块，而非厚重按钮感

### Topbar

- 搜索栏更像命令检索入口
- 右侧账号区与页面标题之间留足空白
- 顶栏从“悬浮白条”改成“轻分隔页眉”

### Page Shell

- 页头增加状态线，例如“Last sync / Today / Active filters”
- 每页都保留统一的上边距与阅读节奏
- 强化大标题和说明的编辑感层次

## Page-by-Page Direction

### Dashboard

目标：像总控日报，不像 KPI 卡片墙。

- 第一屏以“标题 + 当日关键数字带 + 一条运营摘要线”为主
- 第二层展示热门书、热力图、机器人与告警摘要
- 第三层展示书柜运行快照与关键异常
- 用横向数据带替代堆叠卡片

### Books

目标：像图书编辑台。

- 左侧为图书列表与筛选
- 右侧为当前选中图书的编辑检视区
- 分类、标签、状态切换并入统一工作台，而不是分散小卡片

### Inventory

目标：像设备与库存联合作业台。

- 顶部是书柜状态带
- 中部是仓位表 / 记录 / 告警切换
- 右侧或下层保留库存校正编辑区

### Orders

目标：像履约队列而不是订单总览页。

- 把状态筛选、优先级、重试、人工介入做成明显的操作工作流
- 详情页强调时间线、任务状态、异常处理区

### Robots

目标：像调度面板。

- 机器人状态和任务队列是两个互相联动的主区域
- 电量、心跳、当前任务、重分配动作放在同一视野

### Alerts

目标：像告警分诊台。

- 告警与审计日志分别有清晰分区
- 严重度与处理状态视觉区分明确
- 页面气质更安静、更严肃

### Readers

目标：像用户画像工作台。

- 左侧是用户队列与检索
- 右侧是画像、限制、分群、风险标签编辑
- 不做“资料卡片拼盘”，强调结构化编辑

### Recommendation

目标：像运营策划台。

- 推荐位、专题书单、策略权重形成从“内容位”到“机制”的完整链路
- 运营指标区强调趋势和转化，不做浮夸大屏

### Analytics

目标：像分析简报。

- 数字摘要先行
- 趋势、偏好、时段、周转、效率分成清晰版面
- 保持图表与说明之间的留白，让页面更像报告而不是控件堆积

### System

目标：像治理台。

- 配置编辑与角色权限分成两个稳定区域
- 风格更克制、更偏专业管理界面

## Shared Components To Introduce or Refactor

- `PageShell`：重写为统一编辑台页头
- `Sidebar` / `Topbar`：重写视觉与层次
- `StatCard`：升级为“数据带”风格，弱化传统卡片感
- 新增轻量共享组件：
  - `SectionIntro`
  - `MetricStrip`
  - `WorkspacePanel`
  - `InspectorPanel`
  - `SegmentedSection`

## Copy Direction

- 全部页面采用产品 UI 文案，不写营销文案
- 标题直接描述作用，例如“当前系统配置”“机器人任务队列”“库存告警”
- 支撑文案只说明范围、刷新频率、动作影响

## Accessibility and Responsiveness

- 保证桌面端有清晰的双栏/三栏结构
- 移动端改成单列折叠，不保留复杂并列布局
- 颜色对比优先保证文字和状态可读性
- 动画默认短时长，可在弱性能设备上自然降级

## Constraints

- 不重写业务逻辑，不改变 API 契约
- 不删除现有测试覆盖的核心功能
- 页面重设计必须在现有 `React + Vite + Tailwind + Framer Motion` 栈内完成

## Success Criteria

- 首页不再呈现通用 KPI 卡片墙观感
- 所有页面都能通过“标题 + 结构 + 重点区”一眼识别其职责
- 整体风格统一，但每页有明确的版式性格
- 保留现有全部功能入口与权限控制
- 构建、测试通过，不引入明显回归
