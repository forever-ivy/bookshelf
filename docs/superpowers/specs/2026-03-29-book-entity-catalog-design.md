# 图书实体目录二级视图设计

**背景**

当前图书管理页把“实体”做成了表格内行展开。这个方案虽然能显示副本位置和库存状态，但交互层级弱，列表列里会出现重复的“展开实体”按钮，不符合“点进某本书查看其实体目录”的后台心智。

**目标**

把图书页的实体查看改成工作区内二级视图：

- 一级仍然是图书列表
- 点某本书后进入这本书的“实体目录”子视图
- 子视图同时展示书目本身的信息和该书全部实体副本的位置/状态
- 返回列表后保留原筛选条件和分页
- 当前视图状态同步到 URL，刷新后仍可定位到同一本书的实体目录

**交互设计**

图书表第一列按钮从“展开实体”改为“查看实体”。点击后不再展开表格行，而是在“图书列表”工作区内切换到二级视图。

二级视图结构：

1. 顶部返回入口，返回图书列表
2. 当前书目摘要卡，展示书名、作者、分类、上架状态、ISBN、库存汇总、标签、简介
3. 实体目录列表，按副本展示：
   - 副本编号
   - 书柜名称
   - 槽位编号
   - 馆内位置
   - 库存状态
   - 是否可借
   - 更新时间

如果当前书没有实体副本，则在实体目录区域显示空状态提示，而不是保留一个空表格。

**URL 方案**

保留现有图书筛选参数：

- `q`
- `shelf_status`
- `category_id`
- `tab`

新增 `book_id` 作为当前实体目录子视图的定位参数：

- 无 `book_id` 时显示图书列表
- 有 `book_id` 时显示该图书的实体目录视图
- 返回列表时只移除 `book_id`，其他筛选条件保持不变

**实现边界**

- 本轮只做只读实体目录，不在子视图里提供副本状态编辑
- 不新建路由，仍然留在 `BooksPage`
- 保留现有图书编辑侧边栏能力
- 不强制移除 `DataTable` 的通用行展开能力，但图书页不再使用它

**测试要点**

- 图书页从 URL 读取筛选条件仍然生效
- 点击“查看实体”后 URL 写入 `book_id`
- 二级视图展示书目信息和实体目录
- 返回列表后 `book_id` 被移除，原筛选条件仍保留
