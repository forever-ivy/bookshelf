# `HomeRoute` 分时段问候页头设计

**日期：** 2026-03-30

**目标：** 将首页当前单行标题“今晚路径”改为更有设计感的上下结构问候区：上方显示分时段问候语，下方显示当前用户姓名，整体采用用户确认的“极简编辑感”。

## 背景

当前首页通过 `PageShell` 直接渲染单行标题“今晚路径”。用户希望首页顶部更像一个有气质的欢迎区，而不是通用页面标题，且明确要求：

- 文案随时间切换：`早上好 / 中午好 / 下午好 / 晚上好！`
- 采用上下结构
- 下方只显示用户名，不要附加说明文案
- 风格偏“极简编辑感”，依靠字重、留白、间距和细分隔线体现设计感

因此本轮不是简单替换字符串，而是在不影响其他页面的前提下，为首页接入一个可自定义的页头内容区域。

## 范围

- 包含：
  - 首页页头从单行标题改为上下结构问候区
  - 根据当前时间动态生成问候语
  - 优先显示当前登录用户的 `profile.displayName`
  - 为 `PageShell` 增加仅首页使用的自定义 header 内容能力
  - 覆盖首页问候文案和回退文案测试
- 不包含：
  - 改动首页下方推荐、快捷动作、搜索栏等主体内容结构
  - 批量替换其他页面的 header 设计
  - 新增副标题、头像、重卡片、复杂插画或动效

## 已确认的设计方向

用户已确认采用：

- 极简编辑感
- 上方小号分时段问候
- 下方大号用户名
- 通过字重、间距和一条细分隔线提升设计感
- 不增加第二行辅助说明文案

## 方案选择

本轮采用“`PageShell` 增加可选自定义 header 插槽，首页按需传入”的方案。

不直接把首页逻辑硬编码进 `PageShell` 的原因：

- 首页问候区属于页面特例，不适合作为所有页面默认样式
- 其他页面仍然只需要当前的 `headerTitle` 字符串接口
- 用可选插槽能让壳层继续负责布局和间距，页面只负责自己的视觉内容

不采用“继续只传字符串标题，再用样式近似模拟两行”的原因：

- 字符串标题难以表达上下结构、分隔线和不同字号层次
- 后续测试也无法准确区分问候语和用户名两部分内容

## 渲染规则

### 问候语时间规则

- `05:00 - 10:59`：`早上好`
- `11:00 - 13:59`：`中午好`
- `14:00 - 17:59`：`下午好`
- `18:00 - 04:59`：`晚上好！`

说明：

- 仅“晚上好”保留感叹号，和用户给出的目标文案保持一致
- 时间判断使用客户端当前时间即可，本轮不引入时区配置或服务端时间

### 用户名规则

- 优先显示 `useAppSession().profile?.displayName`
- 若为空、缺失或仅空白字符，则回退显示 `同学`

### 视觉结构

- 第一行：小号、较轻的分时段问候语
- 中间：一条极细的装饰分隔线
- 第二行：大号、较重的用户名
- 整体不放入厚重卡片；保留当前页面背景，靠留白和排版建立视觉重心

## 组件设计

### `PageShell`

新增可选 prop：

- `headerContent?: React.ReactNode`

行为规则：

- 当传入 `headerContent` 时，优先渲染该内容
- 现有 `headerTitle` / `headerDescription` 行为保持不变，供其他页面继续使用
- `showBackButton`、`backgroundDecoration`、滚动和安全区逻辑不变

这样可以把变更控制在最小范围内，避免让首页特殊样式影响其他路由。

### 首页 `HomeRoute`

首页新增一个小型问候区视图：

- 负责生成问候语文本
- 负责读取并整理用户显示名
- 负责定义极简编辑感样式
- 通过 `PageShell` 的 `headerContent` 注入顶部

首页原本传入的 `headerTitle={homeHero.title}` 将不再用于顶部标题展示。

## 文件边界

- `components/navigation/page-shell.tsx`
  - 增加自定义 header 内容入口
- `app/(tabs)/index.tsx`
  - 构建首页问候区并接入 `PageShell`
- `__tests__/app/home-route-marker-highlight.test.tsx`
  - 更新首页头部断言，改为验证问候语与用户名
- `__tests__/ui-shell.test.tsx`
  - 更新首页壳层断言，移除对“今晚路径”的依赖

如有必要，可补一个更聚焦的首页问候时间规则测试，但应优先在现有首页测试里扩展断言，避免新增过多冗余覆盖。

## 测试策略

优先覆盖以下行为：

- 首页默认能渲染用户姓名
- 当 `profile.displayName` 缺失时，显示 `同学`
- 在不同时段下，首页分别显示正确问候语
- 首页不再显示旧标题“今晚路径”
- 现有 marker highlight 与快捷动作逻辑不受这次头部改动影响

## 风险与控制

- 风险：`PageShell` 新增插槽后影响其他页面头部布局
  - 控制：新 prop 设计为可选，默认分支保持当前行为
- 风险：时间判断测试不稳定
  - 控制：在测试中 mock 系统时间，分别覆盖至少一个白天时段和一个夜晚时段
- 风险：用户名为空时顶部出现空白
  - 控制：统一走 `trim()` 后回退到 `同学`

## 交付顺序

1. 为 `PageShell` 增加可选 `headerContent`
2. 在首页实现分时段问候区并接入用户名回退逻辑
3. 更新首页和壳层相关测试
4. 手动检查首页顶部层次、留白和旧标题移除情况
